#### ☁️ eMart Microservices Deployment  
📌 LinkedIn: [View Project](https://www.linkedin.com/in/tolulope-olalere/details/projects/)<br>

**Project Summary**<br>
eMart is a modern, scalable e-commerce web application designed to deliver seamless online shopping experiences. Built using a **microservices architecture**, the application separates concerns across multiple containers and services to improve scalability, maintainability, and performance.<br>

**Role & Responsibilities**<br>
As a DevOps Engineer, I managed the deployment of the application by **containerizing and orchestrating six independent services**, each handling specific responsibilities:<br>
- **client** – Frontend interface delivering a responsive user experience<br>
- **webapp** – Core application logic and user-facing functionality<br>
- **api** – Backend service exposing RESTful endpoints for product, user, and order management<br>
- **nginx** – Reverse proxy and load balancer for efficient request routing<br>
- **emongo** – MongoDB container managing non-relational data storage<br>
- **emartdb** – Secondary database container supporting structured data<br>

**Architecture & Tech Stack**<br>
- Frontend: React.js / HTML / CSS<br>
- Backend API: RESTful Web API<br>
- Database: MongoDB<br>
- Reverse Proxy & Load Balancer: NGINX<br>
- Infrastructure: Dockerized microservices in isolated containers<br>

**Key Tools & Technologies**<br>
Docker | Docker Compose | MongoDB | NGINX | RESTful API | Git | Microservices<br>

**Impact**<br>
- Delivered a highly scalable and maintainable microservices platform<br>
- Improved deployment consistency and reliability<br>
- Enabled independent scaling of individual services<br>
